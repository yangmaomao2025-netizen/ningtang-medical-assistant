# 宁唐医学助手 - 产品需求说明书 v1.0

> 来源：PDF 文件，2026-03-23 收到

## 1. 项目概述

宁唐医学助手是一款基于大语言模型的智能医学问答与辅助诊断系统，通过OpenClaw平台开发，提供跨平台一致的医学AI服务。系统旨在帮助医疗从业者提升诊断效率，为患者提供初步健康咨询，同时支持医学教育和科研文献分析。

## 2. 产品定位

- **核心定位**：专业的医学知识库 + 智能诊断辅助
- **差异化价值**：多模态医学分析 + 跨平台无缝体验
- **品牌调性**：专业、可信赖、温暖

## 3. 目标用户

- 初级医师与实习医生：辅助诊断决策
- 医学专业学生：学习与病例分析
- 护理人员：护理方案与健康指导
- 患者及家属：初步健康咨询（不替代正式诊疗）
- 医学研究人员：文献检索与分析

## 4. 功能需求

### 4.1 核心功能

- **医学问答**：基于权威医学知识库的专业问答
- **症状分析**：根据患者描述提供鉴别诊断建议
- **检验解读**：辅助分析实验室检查结果
- **药物查询**：药品相互作用、禁忌症查询
- **医学文献**：PubMed/万方检索与摘要生成

### 4.2 AI增强功能

- **智能分诊**：评估症状紧急程度建议就诊科室
- **病历生成**：自动生成标准格式病历模板
- **随访提醒**：基于诊断的智能随访计划
- **多轮对话**：支持追问和上下文理解
- **语音输入**：支持语音转文字医学术语

## 5. 非功能需求

- **安全性**：符合HIPAA/中国医疗数据安全要求，敏感信息加密存储
- **准确性**：医学内容需标注置信度，明确提示仅供参考
- **响应速度**：常规查询小于2秒，复杂分析小于10秒
- **可用性**：99.9%服务可用性
- **合规性**：符合医疗器械软件分类要求

## 6. 技术架构

- **后端框架**：OpenClaw + Node.js
- **AI模型**：MiniMax / Kimi / GLM 多模型切换
- **数据存储**：加密MongoDB + Redis缓存
- **医学知识库**：定期更新的医学文献库
- **部署方式**：K8s容器化，支持多地域部署

## 7. 多端适配

- **Web端**：响应式设计，支持Chrome/Edge/Safari/Firefox
- **Windows桌面**：Electron封装，离线缓存支持
- **macOS桌面**：Electron封装，Touch Bar适配
- **Android**：原生开发（Kotlin），Play商店分发
- **iOS**：原生开发（Swift），App Store分发

## 10. 医学知识库模块

### 10.1 知识库分类

医学知识库分为两大类：

**文献向量数据库：**
- 用于文档检索问答（RAG）
- 存储医学文献、指南、教材等非结构化文档
- 支持全文检索和语义相似度匹配

**知识图谱：**
- 用于关系推理和复杂查询
- 存储实体（疾病、症状、药品、检查项目）及关系
- 支持多跳推理查询（如：药品→适应症→禁忌症）

### 10.2 知识库数据来源

**人工维护：**
- 管理员上传医学文档（PDF、Word、Excel）
- 手动编辑知识图谱实体和关系
- 审核机器人自动抓取的内容

**机器人自动采集：**
- 爬取权威医学网站（WHO、NIH、FDA等）
- 定时同步 PubMed 文献摘要
- 下载并解析医学数据库更新

**文件导入：**
- 支持批量上传医学指南、专家共识等文档
- 支持导入 CSV/JSON 格式的结构化数据

### 10.3 知识库组件

**文献向量数据库：**
- 推荐：Milvus / Qdrant / Chroma
- 存储医学文献的向量嵌入
- 支持语义检索和相似度匹配
- 用于 RAG 问答增强

**知识图谱：**
- 推荐：Neo4j / TuGraph
- 节点类型：疾病、症状、药品、检查、手术、科室等
- 边关系：治疗、副作用、禁忌、诊断依据等
- 支持多跳查询和路径推理

### 10.4 专业医学数据库

**药品数据库：**
- 药品说明书、适应症、禁忌症、相互作用
- 药品规格、用法用量、不良反应

**检验参考值库：**
- 检验项目名称、参考范围、临床意义
- 异常结果解读指南

**文献库对接：**
- PubMed：文献摘要检索和摘要生成
- 万方/知网：中文医学文献（可选）
- UpToDate / DynaMed：临床指南（可选）
