## 1. 项目概述

### 1.1 项目背景

宁唐医学助手是一款基于大语言模型的智能医学问答系统，通过 OpenClaw 平台开发，提供跨平台一致的医学 AI 服务。

**核心定位**：OpenClaw 医学问答插件 + 统一后台管理系统 + 多端客户端

#### 3.2.2 交互流程

```
用户输入问题
     ↓
意图识别（判断是否医学相关）
     ↓
构造 Prompt（注入系统提示词 + 上下文）
     ↓
调用 LLM（根据选择的模型）
     ↓
置信度评估（基于回答质量/确定性）
     ↓
添加免责声明 + 紧急提示（如有）
     ↓
流式返回给用户
```

#### 3.2.3 核心算法

**置信度评估策略：**

```typescript
interface ConfidenceResult {
  level: 'high' | 'medium' | 'low';
  score: number;        // 0-1
  reasons: string[];
}

// 评估因素：
// 1. 回答长度（过短可能不确定）
// 2. 不确定性词汇（可能/也许/不确定）
// 3. 专业术语使用率
// 4. 引用来源数量
```

**紧急症状检测：**

```typescript
const URGENT_KEYWORDS = [
  '胸痛', '呼吸困难', '大出血', '休克', '昏迷',
  '窒息', '中毒', '过敏性休克', '心梗', '中风'
  // ...
];

function detectUrgent(content: string): boolean {
  return URGENT_KEYWORDS.some(keyword => content.includes(keyword));
}
```

### 6.1 认证与授权

```
┌─────────────────────────────────────────────────────────────────┐
│                      认证授权流程                                  │
├─────────────────────────────────────────────────────────────────┤
```

### 6.2 安全措施

| 安全措施 | 说明 |
|----------|------|
| **传输加密** | HTTPS/TLS 1.3 全链路加密 |
| **API Key 安全** | BCrypt 哈希存储，不明文传输 |
| **JWT Token** | RS256 签名，短期有效 + Refresh Token |
| **SSO 安全** | SAML 签名验证，OAuth State 参数防 CSRF |
| **输入过滤** | XSS/SQL 注入防护 |
| **限流** | API 限流防止滥用 |
| **审计日志** | 所有敏感操作记录日志 |
| **数据加密** | MongoDB AES-256 加密存储 |

|---------------|------|---------|------|------|

| **operator** | ✅ | ✅ | - | - |
| **readonly** | - | ✅ | - | - |
