### 3.3 消息处理流程

```
客户端消息
     ↓
OpenClaw Gateway 接收
     ↓
医学助手通道插件
     ↓
┌─────────────────────────────┐

│ 2. 验证 JWT Token          │

     ↓
```

## 5. 数据模型

### 5.1 核心实体

```typescript
// 用户
interface User {
  id: string;
  username: string;
  email: string;
  phone?: string;
  passwordHash: string;
  status: 'active' | 'disabled';
  createdAt: Date;
  updatedAt: Date;
}

// 机器人
interface Bot {
  id: string;
  appId: string;              // App ID
  appSecretHash: string;        // Secret 哈希
  appSecretPrefix: string;      // Secret 前缀
  name: string;
  description: string;
  avatar?: string;
  status: 'active' | 'inactive';
  
  // 端配置
  platforms: {
    web: boolean;
    macos: boolean;
    android: boolean;
  };
  
  // OpenClaw 配置
  agentId?: string;
  
  createdBy: string;
  createdAt: Date;
  updatedAt: Date;
}

// 用户-机器人权限
interface BotPermission {
  id: string;
  userId: string;
  botId: string;
  level: 'admin' | 'operator' | 'readonly';
  
  // 端权限
  platforms: {
    web: boolean;
    macos: boolean;
    android: boolean;
  };
  
  expiresAt?: Date;
  grantedBy: string;
  grantedAt: Date;
}
```

## 7. 安全设计

### 7.1 认证流程

```
用户登录
     ↓
机器人平台 API 验证
     ↓
返回 JWT Token
     ↓
客户端携带 Token 访问 OpenClaw Gateway
     ↓
通道插件验证 Token
     ↓
检查用户-机器人权限
     ↓
检查端权限
     ↓
处理消息
```

### 7.2 安全措施

| 措施 | 说明 |
|------|------|
| JWT Token | 用户身份验证 |
| App Secret | 机器人访问密钥 |
| HTTPS | 传输加密 |
| 端权限控制 | 按端启用/禁用 |
| 权限级别 | admin/operator/readonly |
| 消息签名 | 防篡改 |
