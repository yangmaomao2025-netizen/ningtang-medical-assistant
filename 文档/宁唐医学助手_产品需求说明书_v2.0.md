# 宁唐医学助手 - 产品需求说明书 v2.0

> 版本：v2.0
> 日期：2026-03-24

---

## 1. 项目概述

宁唐医学助手是一款基于 OpenClaw 平台开发的医学问答消息通道插件，通过 OpenClaw Gateway 实现跨平台一致的医学 AI 服务。

---

## 2. 系统架构

```
客户端 (Web/macOS/Android) → 消息插件 → Gateway → Agent → AI 大模型
```

- **客户端**：通过 WebSocket 连接消息插件
- **消息插件**：WebSocket 消息路由（核心）
- **Gateway**：OpenClaw 消息路由
- **Agent**：OpenClaw AI Agent
- **AI 大模型**：MiniMax / Kimi / GLM

---

## 3. 产品组成

### 3.1 机器人应用平台

| 模块 | 说明 |
|------|------|
| 用户管理 | 用户体系、登录验证 |
| 机器人管理 | 创建机器人、分配 App ID / App Secret |
| 权限配置 | 用户-机器人权限绑定 |
| 端配置 | Web / macOS / Android 是否启用 |

### 3.2 三端客户端

| 端 | 技术方案 | 连接方式 |
|-----|---------|---------|
| Web | React | WebSocket |
| macOS | Electron | WebSocket |
| Android | Kotlin | WebSocket |

**核心功能：Chat 聊天互动**

### 3.3 OpenClaw 通道插件

| 模块 | 说明 |
|------|------|
| WebSocket 服务 | 接收客户端 WebSocket 连接（核心） |
| 消息接收 | 接收客户端消息 |
| Token 验证 | 验证用户 JWT |
| 消息转发 | 转发到 OpenClaw Agent |
| 响应推送 | AI 回复通过 WebSocket 推送回客户端 |

## 4. Chat 聊天互动功能（详细）

### 4.1 客户端流程

```
登录 → 机器人列表（有权限的） → 点击机器人 → 聊天互动
```

### 4.2 消息功能

- 文本消息发送
- AI 回复展示（流式输出）
- 消息时间戳
- 发送状态（发送中/成功/失败）
- 加载动画（请稍后...）
- 消息重发

### 4.3 多模态消息

| 类型 | 说明 |
|------|------|
| 文本 | 文字消息 |
| 图片 | 上传图片 |
| 文件 | PDF、Word 等附件 |
| 语音 | 语音消息（可选） |

### 4.4 AI 回复功能

- 医学问答（基于 LLM）
- 流式输出（打字机效果）
- 置信度标注（高/中/低）
- 免责声明
- 紧急提示
- 建议追问

### 4.5 用户功能

- 客户端登录
- 密码修改
- 头像维护（自己头像、机器人头像）

### 4.6 对话功能

- 新建对话
- 对话列表
- 对话切换
- 对话标题（自动/手动）
- 对话搜索
- 对话删除

### 4.7 连接状态

- 在线状态显示
- 自动重连
- 离线缓存

---

## 5. 数据流

```
用户发送消息
    ↓
客户端携带 JWT Token
    ↓
OpenClaw Gateway
    ↓
医学助手通道插件（验证 Token）
    ↓
转发到 OpenClaw Agent
    ↓
AI 大模型处理
    ↓
响应原路返回
```

---

## 6. 安全需求

| 措施 | 说明 |
|------|------|
| JWT Token | 用户身份令牌 |
| App Secret | 机器人访问密钥 |
| HTTPS | 传输加密 |
| 用户-机器人绑定 | 只有绑定用户才能访问 |
| 端权限控制 | 按端启用/禁用 |
