## 1. 项目概述

### 1.1 项目背景

宁唐医学助手是一款基于 OpenClaw 平台开发的医学问答通道插件，通过 OpenClaw Gateway 实现跨平台一致的医学 AI 服务。用户通过三端客户端（Web / macOS / Android）访问服务，消息通过 OpenClaw Gateway 路由到医学助手通道插件处理。

### 1.2 产品定位

- **核心定位**：OpenClaw 消息通道插件 + 机器人应用平台
- **差异化价值**：专业医学知识 + 多端一致体验
- **品牌调性**：专业、可信赖、温暖

## 4. 数据流

### 4.1 消息流程

```
用户发送消息
     ↓
客户端（携带 JWT Token）
     ↓
OpenClaw Gateway
     ↓
医学助手通道插件（验证 Token）
     ↓
转发到 OpenClaw Agent
     ↓
AI 大模型处理
     ↓
响应原路返回
```

### 4.2 认证流程

```
客户端登录
     ↓
机器人平台 API（验证账号）
     ↓
返回 JWT Token
     ↓
客户端携带 Token 访问
     ↓
OpenClaw Gateway 验证 Token
     ↓
路由到医学助手通道
```

## 5. 安全需求

### 5.1 认证安全

| 措施 | 说明 |
|------|------|
| JWT Token | 用户身份令牌 |
| App Secret | 机器人访问密钥 |
| HTTPS | 传输加密 |
| Token 过期 | 定期刷新 |

### 5.2 访问控制

| 措施 | 说明 |
|------|------|
| 用户-机器人绑定 | 只有绑定用户才能访问 |
| 端权限控制 | 按端启用/禁用 |
| 权限级别 | admin/operator/readonly |
